# חידון PWA בעברית (RTL)

אפליקציית חידון דינמית שמקבלת קובץ JSON, עם תמיכה מלאה בעברית ו-RTL, כולל עבודה כ-PWA (מתאים לפריסה ב-GitHub Pages).

## הרצה מקומית

אפשר לפתוח את `index.html` ישירות בדפדפן, אבל עבור service worker מומלץ להריץ שרת סטטי.

דוגמה עם PowerShell (אם מותקן Python):

```powershell
python -m http.server 8080
```

ואז לפתוח: `http://localhost:8080`

## פריסה ב-GitHub Pages

1. דוחפים את כל הקבצים לריפו.
2. ב-GitHub: `Settings > Pages`.
3. בוחרים `Deploy from a branch`.
4. בוחרים את הסניף (`main`) ואת התיקייה (`/root`).
5. שומרים וממתינים לפריסה.

`manifest.webmanifest` ו-`sw.js` כבר מוגדרים לשימוש יחסי (`./`) כך שיעבדו היטב גם ב-GitHub Pages.

## פורמט JSON לקובץ חידון

יש דוגמה מלאה בקובץ: `data/default-quiz-he.json`.

מבנה נדרש:

```json
{
  "title": "שם החידון",
  "questions": [
    {
      "question": "טקסט השאלה",
      "answers": ["תשובה 1", "תשובה 2", "תשובה 3", "תשובה 4"],
      "image": "./data/images/example.svg",
      "correctIndex": 0
    }
  ]
}
```

כללים:

- `title`: מחרוזת, שם החידון.
- `questions`: מערך של שאלות (לפחות שאלה אחת).
- `question`: מחרוזת עם טקסט השאלה.
- `answers`: מערך מחרוזות עם לפחות 2 תשובות.
- `image` (אופציונלי): שם/נתיב לקובץ תמונה שיוצג עבור השאלה.
- `correctIndex`: אינדקס מספרי של התשובה הנכונה בתוך `answers` (מתחיל ב-0).

## התנהגות האפליקציה

- מסך ראשון: ברוכים הבאים + טעינת קובץ או התחלה עם ברירת מחדל.
- מסך ביניים: שם החידון בתוך באנר וכפתור התחלה.
- מסך שאלה: תשובות בסידור וצבעים משתנים.
  - תשובה שגויה: מוצג `X` לשתי שניות וחוזר לטקסט.
  - תשובה נכונה: מוצג `V` לשנייה, הניקוד עולה ב-2 ועוברים לשאלה הבאה.
- מסך סיכום: מציג כמה תשובות נכונות היו מתוך הסך הכולל.
- קישורי `התחלה מחדש` ו-`טעינה מחדש` מאפסים ניקוד בהתאם לזרימה.
